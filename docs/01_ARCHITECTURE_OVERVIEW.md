# 01 — نظرة عامة على المعمارية (Architecture Overview)

## 1. الطبقات الرئيسية

```
┌─────────────────────────────────────────────────────────────────┐
│                    Dashboard (Streamlit)                          │
│  إدارة الكلمات المفتاحية | جدولة | إعدادات LLM | مراقبة التوكينز  │

                                    │
                                    │ HTTP / REST API
                                    ▼
┌─────────────────────────────────────────────────────────────────┐

                                    ▼
```

## 2. مسؤوليات كل طبقة

### 2.1 واجهة التحكم (Streamlit Dashboard)

| الوظيفة | الوصف |
|---------|--------|
| إدارة الكلمات المفتاحية | إضافة / تعديل / حذف / استيراد keywords |
| جدولة المقالات | تقويم النشر واختيار أوقات النشر |
| اختيار نموذج LLM | من قائمة OpenRouter (GPT-4, Claude, Mixtral, إلخ) |
| متابعة حالة المقالات | مسودة، قيد التوليد، منشور، فشل |
| مراقبة استهلاك التوكينز | يومي / شهري / لكل مقال |
| التحليلات | عدد المقالات، معدل النشر، جودة SEO |

### 2.2 طبقة الخدمات (FastAPI Backend)

| الوظيفة | الوصف |
|---------|--------|
| API للداشبورد | جميع عمليات CRUD والمعاملات عبر REST |
| إدارة قاعدة البيانات | مقالات، كلمات مفتاحية، جدولة، توكينز |
| تشغيل الـ AI workflows | استدعاء LangGraph وتشغيل الـ pipeline |
| جدولة النشر | APScheduler لمهام النشر والتوليد |
| الاتصال بـ WordPress | نشر المقالات عبر REST API |

### 2.3 محرك الذكاء الاصطناعي (LangGraph + OpenRouter)

| الوظيفة | الوصف |
|---------|--------|
| تنفيذ الـ pipeline | تنفيذ العقد بالترتيب مع تمرير الحالة |
| استدعاء LLM | عبر OpenRouter (نموذج قابل للتغيير) |
| تتبع التوكينز | تسجيل الاستهلاك لكل استدعاء |
| التعامل مع الأخطاء | إعادة المحاولة، fallback، تسجيل |

---

## 3. تدفق البيانات (Data Flow)

1. **المستخدم** يضيف keyword أو يختار مقالة من الجدول من الـ Dashboard.
2. **Backend** يستقبل الطلب ويستدعي الـ AI Engine (LangGraph).
3. **AI Engine** يمر عبر: Keyword → Brief → Article → Quality → Duplicate → SEO.
4. **Backend** يحفظ المقال في DB ويُحدّث الحالة.
5. **Scheduler** (أو المستخدم) يطلق عملية النشر إلى WordPress.
6. **WordPress Service** يرسل `POST /wp/v2/posts` ويُحدّث حالة المقال إلى "منشور".

## 4. التقنيات المستخدمة

| المكون | التقنية |
|--------|----------|
| Dashboard | Streamlit |
| Backend API | FastAPI |
| AI Workflow | LangGraph |
| LLM Gateway | OpenRouter |
| قاعدة البيانات | SQLite (تطوير) / PostgreSQL (إنتاج مقترح) |
| ORM | SQLAlchemy (مقترح) |
| الجدولة | APScheduler |
| النشر | WordPress REST API |
