# 06 — مواصفات Dashboard (Streamlit)

## 1. الهيكل العام

- تطبيق واحد: `dashboard/streamlit_app.py`.
- تنقل جانبي (sidebar) بين الصفحات.
- الاتصال بالـ Backend عبر `requests` أو `httpx` إلى عنوان FastAPI (مثلاً `http://localhost:8000`).

---

## 2. الصفحات

### الصفحة 1 — Content Calendar

| العنصر | الوصف |
|--------|--------|
| التقويم | عرض مقالات مجدولة للنشر حسب التاريخ (من API `/scheduler/jobs` و/أو `/articles`) |
| إضافة حدث | زر أو نموذج لجدولة مقال أو توليد جديد في تاريخ معين |
| فلترة | حسب الحالة (مسودة، منشور، مجدول) |

### الصفحة 2 — Keywords Manager

| العنصر | الوصف |
|--------|--------|
| جدول الكلمات | قائمة من GET `/keywords` مع إمكانية بحث وفلترة |
| إضافة كلمة | نموذج لإضافة كلمة واحدة أو عدة كلمات |
| استيراد | رفع ملف (csv/txt) واستدعاء POST `/keywords/import` |
| تعديل/حذف | أزرار أو قائمة إجراءات لكل صف |

### الصفحة 3 — Articles Manager

| العنصر | الوصف |
|--------|--------|
| جدول المقالات | من GET `/articles` (عنوان، حالة، تاريخ، كلمة مفتاحية) |
| فلترة | حسب الحالة (مسودة، قيد التوليد، منشور، فشل) |
| تشغيل التوليد | نموذج: اختيار keyword وإعدادات ثم استدعاء POST `/articles/generate` |
| معاينة/تعديل | فتح تفاصيل مقال وتحرير إن أمكن |
| نشر | زر "نشر" يستدعي POST `/articles/{id}/publish` |

### الصفحة 4 — Token Usage

| العنصر | الوصف |
|--------|--------|
| فترة | اختيار (يوم / أسبوع / شهر) أو نطاق تاريخ |
| رسم بياني | استهلاك التوكينز عبر الزمن (من GET `/usage/tokens`) |
| جدول تفصيلي | استهلاك لكل مقال (من GET `/usage/by-article`) إن وُجد |
| إجمالي التكلفة | عرض إجمالي التكلفة للفترة |

### الصفحة 5 — AI Settings

| العنصر | الوصف |
|--------|--------|
| نموذج LLM | قائمة من نماذج OpenRouter المدعومة (من config أو API) |
| Temperature | slider (0–1 أو 0–2 حسب النموذج) |
| Max tokens | حقل رقمي |
| حفظ | استدعاء PATCH `/settings/ai` لتحديث الإعدادات |

---

## 3. الإعدادات المشتركة

- **Backend URL:** متغير في بداية الملف أو من `.env` (مثلاً `BACKEND_URL`).
- **عرض الأخطاء:** عرض رسائل الخطأ من الـ API في الـ Dashboard (st.error أو toast).
- **تحديث تلقائي:** اختياري — إعادة تشغيل الاستعلام عند تغيير الفلاتر أو الفترة.

---

## 4. ترتيب التنفيذ الموصى به

1. شريط جانبي مع اختيار الصفحات.
2. صفحة Articles Manager (الأساسية).
3. صفحة Keywords Manager.
4. صفحة AI Settings.
5. صفحة Token Usage.
6. صفحة Content Calendar (تعتمد على Scheduler API).
